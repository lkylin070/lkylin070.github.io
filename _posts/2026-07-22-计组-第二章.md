# 计组-第二章

# 总线系统

## 考点1
### 总线

总线是连接多个系统功能部件之间进行数据传输的公共通路，**实现地址､数据和控制信息的交换**

**总线的两个特点**：分时､共享

1. 分时：是指同一时刻只允许一个部件向总线发送信息，只能分时发送

2. 共享：总线上可以挂多个部件，各个部件之间可以通过这组线路分时享用，多个部件可以同时从总线上接送相同的信息
串联电路中，电阻越大，分到的电压越大；电阻越小，分到的电压越小。

**总线设备**

1. 主设备：获得总线控制权的设备

2. 从设备：被主设备访问的设备，只响应从主设备发来的各种总线命令

**总线的分类**

1. 片内总线：芯片内部的总线，是CPU芯片内部寄存器与寄存器之间、寄存器与AUL之间的公共连接线

2. 系统总线：CPU、主存、IO接口之间相互连接的总线，按总线的传输信息内容的不同，有可以分为3类：

    - 数据总线：双向，与机器字长、存储字长有关

    - 地址总线：单向，与存储地址、IO地址有关

    - 控制总线：每根线都是单向的，有出，有入

3. 通信总线：用于计算机系统之间，或计算机系统与与其他系统之间的通信（控制仪表，移动通信）

    - 按时序控制方式，分为同步总线和异步总线

    - 按数据传输方式，分为并行总线和串行总线

## 考点2

### 总线的组成与特性

**总线的内部结构**

总线由地址线、数据线、控制线组成

1. 仲裁总线：总线请求线和总线授权线

2. 中断和同步总线：用于处理优先级的中断操作，包括中断请求线和中断认可线

3. 公用线：包括时钟信号线、电源线、地址线、系统复位线

**总线的特性**

1. 物理特性：总线的物理连接方式（根数、插头、插座形式、引脚排列方式）

2. 功能特性：每根线的功能

3. 电气特性：每根线上信号的传递方向及有效电平范围

4. 时间特性：规定了每根总线在什么时间有效

## 考点3 总线性能指标

1. 总线时钟周期：机器的时钟周期

2. 总线的时钟频率：机器的时钟频率

3. 总线的传输周期：是指一次总线操作所需的时间，简称总线周期，总线传输周期由n个总线时钟周期构成

4. 总线的工作频率：总线上各种操作的频率，为总线周期的倒数，如总线周期=N个时钟周期，则总线工作频率f=时钟频率/N

5. 总线宽度：总线位宽，是总线上同时能够传输的数据位数，通常是指数据总线的根数

6. 总线带宽：单位时间内总线上传输的最大字节数MB/s，总线带宽=总线宽度*总线频率，例如：33MHz，32位（4个字节）总线带宽 = 33*4=132MB/s

7. 总线复用：一种信号在不同时间传输不同的信息，例如地址线和数据线 的复用

8. 信号线数：地址线、数据线和控制线的总和

## 考点3 总线传送方式

1. 串行传送：使用一条传输线，用脉冲传送，主要优点，只需要一条传输线，成本低，速度慢

2. 并行传输：每一数据位传输需要一条传输线，一般采用电位传送，系统总线上的传送的信息必须采用并行

3. 分时传送：即总线的分时复用

## 考点5 总线接口

1. 接口是CPU和主存、外设之间通过总线进行连接的逻辑部件

2. 接口典型功能：控制，缓冲，转换、整理、程序中断等

3. 一个适配器（接口）的两个接口，一个和系统总线相连，并行，一个和设备相连，并行或者串行

## 考点6 总线结构

### 单线结构

1. 总线结构把CPU、主存、IO设备都挂在一组总线上，运行IO设备之间、IO设备与主存之间直接交换信息

2. CPU与主存、CPU与外设之间可以直接进行信息交换

3. 系统总线按传送信息不同分为，地址总线，数据总线，控制总线

4. 优点：结构简单，成本低，易于接入新的设备。

5. 缺点：带宽低、负载重

6. 要求连接到总线上的逻辑部件必须高速运行

### 双总线结构

1. 一条主存总线，用于CPU、主存和通道之间进行数据传送，另一条IO总线，用多个外部设备与通道之间进行数据传送

2. 优点：把较低的IO设备从单总线上分离出来，实现存储器总线和IO总线分离

3. 缺点：需要增加通道等硬件设备

### 三线结构

1. 主存总线，IO总线，直接内存访问总线DMA总线

2. 主存总线用于CPU和内存之间传送地址，数据，控制信息，IO总线用于CPU和各类外设之间通信，DMA总线用于内存和高速外设之间直接传送数据

3. 优点：提高了IO设备的性能，提高系统的吞吐量

4. 缺点：系统工作效率较低

### 三总线结构2

1. 新增了一条扩展总线和局部总线

2. 局部总线把CPU和Cache相连，Cache通过系统总线和主存连接，这样CPU只和Cache通信，当Cache中没有数据时，Cache和主存通信取出数据

3. 所有的IO设备通过扩展总线和系统总线与主存通信

### 四总线

新增了一条高速总线，负责连接一些高速的IO设备

### 总线的连接方式

1. 高速CPU总线，CPU和Cache之间采用

2. 系统总线：主存连接在其上

3. 高速总线可以连接一些，高速外设

4. 通过桥，CPU总线，系统总线和高速总线彼此相连，桥是一种具有缓冲、转换、控制功能的逻辑电路

### 总线结构举例

1. 传统的微型机总线结构：实际山是处理芯片的延伸，是处理器和IO设备适配器之间的通道，按功能分为地址、数据、控制线

2. 大多数计算机采用了分层次的多总线结构：VL-BUS局部总线结构，PCI总线结构，多层PCI总线结构

### Host总线

1. 有CPU总线、系统总线、主存总线、前端总线等，Host总线不仅连接主存，还可以连接多个CPU

2. Host总线是连接北桥芯片与CPU之间的信息通路，总线上有L2级的Cache，主存和Cache的控制器芯片，后者可以用来管理CPU对主存和Cache的存取操作，CPU拥有Host总线的控制权

# 总线仲裁

## 考点7 集中式仲裁

### 总线仲裁的概念

1. 连接到总线上的功能模块用主动和被动两种形态，主方可以启动一个总线周期，而从方只能响应主方请求，每次操作**只能有一个主方**，可以要**多个从方**

2. 为解决多个功能模块争用总线的问题，必须设置总线仲裁部件

3. 按照总线仲裁电脑位置不同，仲裁方式分为集中式和分布式

4. 集中总裁：链式查询，计数器定时查询，独立请求

### 链表查询方式

1. BG行地从一个IO接口到下一个IO接口，如果BG到达的接口**有总线请求**，BG信号就不再往下传，然后接口**获得了总线使用权**，并建立**总线忙的BS信号**

2. 离中央仲裁器最近的设备具有最高优先权，离总线控制器越远，优先权越低

3. **优点**：只用很少几根线就能按一定优先次序实现总线控制，很容易扩充设备

4. 缺点：**对询问链的电路敏感，优先级固定**

### 计数器定时查询方式

1. 设备--通过BR线发出请求-->中央仲裁器--如果BS线为0-->计数器开始计数--当有设备的地址=计数值-->给改设备置"1"B线，获得总线使用权

2. 每次计数可以从“0”开始，也可以从中止点开始，如果从0开始就是链表查询方式了，如果从中止点开始，每个设备的使用总线的优先级相等

3. 优点：**计数器的初值也可以用程序来设置**，可以方便的改变优先次序

4. 缺点是增加了控制线，控制比链式复杂

### 独立请求方式

1. 每个共享总线的设备有一对总线请求线BR\_i和总线授权线BG\_i。当设备要求i使用总线，则发出信号

2. 总线仲裁器有一个排队电路，根据一定的优先次序决定首先响应哪个设备请求，给设备以授权信号BG\_i

3. 优点：响应时间快，对优先次序的控制相当灵活，它可以预先固定，也可以通过程序来改变优先次序，还可以用屏蔽某个请求的的方法

4. 缺点：控制线数量多，总线控制逻辑复杂

## 考点8 分布式仲裁

1. 不需中央仲裁器，由多个仲裁器竞争使用总线

2. 当它们用总线请求时，把它们唯一的仲裁号，发送到共享的仲裁总线上，每个仲裁器将仲裁总线上得到的号与自己的号进行比较

3. 如果仲裁总线上的号大，它的总线请求不给予响应，并撤销仲裁号

4. 最后，获胜者的仲裁号留在仲裁总线上，分布仲裁是以优先级仲裁策略为基础

# 总线操作和定时

## 考点9 总线周期

1. 申请分配阶段：传输请求和总线仲裁

2. 寻址阶段：取得了使用权的主模块，通过总线发出命令，寻找本次要访问的模块

3. 传输阶段：主模块和从模块进行数据交换，可单向或者双向

4. 结束阶段：主模块的有关信息从总线上退出

## 考点10 总线定时 同步方式

### 总线定时的作用

总线定时是指在双方交互数据的过程中，需要**时间**上配合关系控制，这种控制称为**总线定时**，实际上是一种协议或者规则，主要有**同步**和**异步**两种基本方式

### 同步方式的概念

1. 同步定时是指**系统采用一个统一的时钟信号来协调发送和接受双方传送的定时关系**

2. 时钟产生相等的时间间隔，每个间隔构成一个总线周期，在一个周期中，发送方和接受方可以进行一次数据传送

3. 每个部件或者设备发送或接受信息都在固定的总线传送周期中，一个总线的传送周期结束，下一个传送周期开始

### 同步方式的特点

1. 优点：传送速度快，有较高的传输速率，总线控制逻辑简单

2. 缺点：主从设备是强制性同步，不能及时进行数据通信的有效性检验，可靠行较差

3. 同步通信使用于总线长度较短及总线所接部件的存取时间比较接近的系统

## 考点11 总线定时 异步方式

### 异步方式的概念

1. 在异步定时方式中，没有统一的时钟，没有固定的时间间隔，完全依靠双方相互制约的握手信号来实现定时控制

2. 主设备提出交换信息的请求信号，经过接口传送到从设备，从设备接受，然后通过接口向主设备发出回答信号

### 异步方式的特点

1. 优点：总线周期长度可变，能保证两个工作速度相差很大的部件或者设备之间可靠的进行信息交换，自动适应时间配合

2. 缺点：比较同步控制方式复杂一些，速度慢一些

### 异步方式的类型

根据请求和回答信号的撤销是否**互锁**，异步定时的方式又分为3种类型

1. 不互锁方式

   主设备发出请求后，不用等接到从设备的回答信号，而是经过一段时间，便撤销请求信号，双方无互锁的关系

2. 半互锁方式

   主设备发出请求后，必须等接到从设备的回答信号，才能撤销请求信号，有互锁的关系

   从设备发出回答信号，不必等待获取主设备的请求信号，而是隔一段时间自动撤销回答信号，不存在互锁关系

3. 全互锁方式

    主设备发出请求后，必须等接到从设备的回答信号，才能撤销请求信号

    从设备发出回答信号，必须等待获取主设备的请求信号，再撤销回答信号。双方存在互锁关系

## 考点12 总线定时 半同步方式

### 半同步方式的概念

1. 半同步方式是同步和异步的结合

2. 同步：发送方用系统时钟前沿发信号，接收方用系统时钟后沿判、识别

3. 异步：允许不同速度的模块和谐工作，增加一条等待响应信号

## 考点13 总线定时 分离式

### 同步、异步、半同步方式特点

一个总线传输周期

1. 主模块发送地址、命令、占用总线

2. 从模块准备数据，不占用总线，总线空闲

3. 从模块向主模块发送数据，占用总线

### 分离式通信的实现

分离式通信充分挖掘系统总线每个瞬间的潜力。一个总线传输周期分为：

1. 子周期1：主模块申请占用总线，使用完后，放弃总线使用权

2. 子周期2：从模块向主模块申请占用总线，把各种信息传送到总线上

### 分离式通信的特点

1. 各模块有权申请占用总线

2. 采用同步方式通信，不等对方回答

3. 各模块准备数据时，不占用总线

4. 总线被占用时，无空闲

# 总线标准

## 考点14 典型总线

1. 总线标准是国际上公布和推荐的

2. 总线标准设计的接口可视为通用接口，在接口的两段，任何一方只需要根据总线标准的要求，完成自身方面的功能，无需对方接口要求

3. 目前流行的总线标准分为**系统标准和设备标准**

4. 不同类型总线的主要区别是总线宽度、带宽、时钟频率、寻址能力、是否支持突发传送等

### 系统总线标准

1. ISA总线：最早出现微型计算机的16位系统总线标准，应用在IBM的AT机上

2. EISA总线：为配合32位CPU而设计的总线扩展标准，EISA对ISA完全兼容

3. VESA总线：32位标准的计算机局部总线，针对多媒体PC要求高速传送活动图像的大量数据应运而生

4. PCI总线：**高性能的32位或64位并行总线**。是专为高度集成的外围部件、扩充插班和处理器/存储器系统而设计的互联机制

5. PCI-Express(PCI-R)：是最新的总线和接口标准，是一种高速串行扩展总线，这个标准将全面取代现行的PCI和AGP标准

### 设备总线标准

1. IDE总线：一种磁盘驱动器并行接口类型，硬盘和光盘通过IDE接口与主板接口连接

2. AGP总线：一种视频接口标准，专用与连接主存和图形存储器，属于局部总线

3. USB总线：通用串行总线，一种连接外部设备的IO总线标准，具有即插即用、热插拔等优点，用很强的连接能力

4. RS-232C总线：美国电子协会推荐的一种串行通信总线标准是用于串行二进制交换的数据终端设备和数据通信设备之间的标准接口

5. SATA：是一种基于行业标准的串行硬件驱动器接口

6. SCSI：是一种用于计算机和智能设备之间（硬盘、软驱、光驱、打印机等）系统级接口的独立处理器标准，SCSI是一直并行接口标准

## 考点15 PCI总线

### PCI总线的概念

1. PCI总线是高性能的32位或者64位总线，是专为高度集成的外围部件、扩充插板和处理器/存储器系统而设计的互联机制

2. 目前常用的PCI适配器用显卡、声卡、网卡等

### PCI总线的特点

1. PCI总线是支持即插即用，是一个与处理器时钟频率无关的高速外围总线，属于局部总线，PCI可以通过桥连接实现多层PCI总线

2. PCI总线用来连接各种高速的PCI设备，是一个与处理器无关的高速外围总线

3. 在PCI设备中不存在DMA（直接存储器传送）的概念，这是因为PCI总线支持无限的猝发式传送

### PCI总线的仲裁方式

1. 采用同步时序协议和集中式仲裁策略，具有自动配置能力

2. PCI设备可以是主设备，也可以是从设备

### 桥

1. 系统允许有多条PCI总线，可以使用Host桥与Host总线相连，也可以用PCI/PCI桥，从而扩充PCI总线负载能力

2. PCI总线体系结构中的三种桥：Host桥是PCI总线控制器，含有中央仲裁器

3. 桥是一个总线转换部件，可以把一条总线的地址空间映射到另一条总线的地址空间上，从而使系统中任意一个总线主设备都能看到同样的一方地址表

4. 桥本身结构可以十分简单，像只有信号缓冲能力和信号电平转换逻辑，也可以相当复杂，像有规程转换、数据快存、装拆数据等
